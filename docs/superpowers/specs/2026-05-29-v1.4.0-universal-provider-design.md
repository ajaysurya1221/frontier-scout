# v1.4.0 — Universal Provider + Implement & Test

Status: approved 2026-05-29. Branch: `feat/v1.4.0-universal-provider`.

## Problem

v1.3.0 shipped a polished TUI but three honest gaps remain:

1. **No provider flexibility.** The live pipeline only ever builds
   `anthropic.Anthropic`. A user with *only* an OpenAI key, *only* the
   Claude Code CLI, or *only* the Codex CLI cannot run a live scout —
   they hit `RuntimeError: ANTHROPIC_API_KEY is required`.
2. **Irrelevant scout results.** AI-tool scan and dependency scan merge
   into one unfiltered table; the score rubric never excludes web
   frameworks, so FastAPI leaks into "AI tools".
3. **No close-the-loop action.** Approving a verdict does nothing to the
   repo. There is no "apply it and prove it works".

Plus: README cost is Anthropic-only and unverified; demo is reachable
from the TUI's recommended actions when it should be a pure CLI affordance.

## Design

### Stream 1 — Provider abstraction (`frontier_scout/providers/`)

Single `LLMProvider` protocol; four backends auto-detected in priority
order: Anthropic key → OpenAI key → Claude Code CLI → Codex CLI.
`policy.toml` / setup-state `provider` pins a choice.

- `base.py` — `LLMProvider` protocol, `ProviderResponse` (Anthropic-shaped
  `.content` blocks + `.usage` + actual `.model`), `ToolUseBlock`,
  `TextBlock`, `Usage`, `ProviderUnavailable`.
- `anthropic_provider.py` — passthrough; returns the raw SDK response so
  every existing extraction site is byte-for-byte unchanged.
- `openai_provider.py` — translates Anthropic tool schema → OpenAI
  function-calling; strips `cache_control`; maps `tool_choice`; rebuilds
  a `ToolUseBlock` from the function call; maps token usage.
- `claude_code_provider.py` / `codex_provider.py` — shell out to the CLI
  in non-interactive mode, embed the tool's JSON schema in the prompt,
  parse JSON-only stdout, validate.
- `resolve_provider()` — detection + pin + clean "no provider → run
  `frontier-scout --demo`" message (never a raw traceback).

Call sites thread a `tier` (`"fast"` | `"deep"`) instead of a hardcoded
model id; the provider maps tier → its own model and reports the actual
model back so the cost ledger attributes spend correctly. `cost_tracker`
gains OpenAI pricing and a defensive lookup (unknown/CLI model → $0).

### Stream 2 — Categorization fix

`dev_tool` rubric explicitly excludes web/app frameworks (FastAPI, Django,
Flask, Express, Next.js, …). A verdict must clear an AI-adoption fit
threshold, not generic popularity. TUI Scout table groups rows by category
so AI tools and dependency upgrades are visually separated.

### Stream 3 — RLAIF loop (`scripts/rlaif.py`)

Opus-as-judge harness, run live with real keys against real repos:
scout → judge scores each verdict for relevance + fit honesty → emits
rubric deltas → re-run, until zero false positives two cycles running.
Hard budget cap $60–70 with a running tally printed each cycle.
Artifact: `demo/rlaif-report.md`.

### Stream 4 — Implement & Test (`frontier_scout/implement.py`)

On approving a verdict with a concrete change recipe: isolated git
worktree → apply change → auto-detect test command (`pytest` / `make test`
/ `npm test`) → run → report pass/fail + diff summary + "what you get" →
offer Keep (merge) / Discard. Hermetic and reversible; reuses Lab
isolation patterns. TUI button + CLI command.

### Stream 5 — `--demo` + TUI cleanup

`frontier-scout --demo` top-level flag aliasing the existing `demo`
subcommand. Drop the `demo_report` recommended action from
`setup_diagnostics.py`. `report.py` server untouched.

### Stream 6 — Cost docs + single-LLM UX + release

README cost rewrite with measured Anthropic + OpenAI numbers and a note
that Claude Code / Codex backends are $0 marginal. Single-provider
onboarding messaging. CHANGELOG, version 1.4.0, clarity sweep, PR,
CodeRabbit, merge, tag, PyPI.

## Non-negotiables

- Suite green at every commit (287 baseline).
- Never spend on a user's key silently; RLAIF prints spend each cycle and
  hard-stops at the cap.
- Lab/implement subprocesses never see real HOME or parent credentials.
- Env secrets used, never echoed.
